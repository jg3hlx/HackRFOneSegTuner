# HackRF One-Seg 受信パイプライン

## 全体処理フロー

```

│  533.143 MHz (ch23), 8.127 Msps, Complex Float32 (I+Q)
│
▼
┌─────────────────────────────┐
│  周波数補正 (Rotator)        │  HackRF水晶の~20PPM誤差を補正

└─────────────┬───────────────┘
              ▼
┌─────────────────────────────┐
│  ローパスフィルタ (LPF)       │  帯域幅 5.8 MHz → ISDB-T 1ch分を抽出
│  Hamming窓 FIR              │  隣接チャンネルの干渉を除去
└─────────────┬───────────────┘
              ▼
┌─────────────────────────────────────────────┐
│  OFDM同期 (ofdm_synchronization)            │
│                                             │

│     シンボル末尾の相関でシンボル境界を検出     │
│                                             │
│  2. 細かい周波数補正: CP位相差から             │
│     残留周波数オフセットを推定・補正           │
│                                             │
│  3. FFT: 8192点 → 周波数領域へ変換           │
│     ┌────────────────────────────────┐      │
│     │ [0...1287][  5617 active  ][1288]│      │
│     │  ゼロ埋め  ← 使うのはここ → ゼロ  │      │
│     └────────────────────────────────┘      │
│                                             │
│  4. チャネル等化: 散乱パイロット(SP)の         │
│     既知値と受信値を比較 → 各キャリアの        │
│     振幅・位相歪みを補正                      │

               ▼

               │

│                            │  2ビットのソフト判定値へ変換

               │
```

## 独自パッチ箇所

**gr-isdbt `ofdm_synchronization_impl.cc`:**
- VOLK境界パディング追加（メモリ安全性）
- 整数周波数オフセット探索範囲: ±10 → ±30 サブキャリア
- 二重補正バグ修正: `d_total_freq_error` から整数オフセット成分を除去
- TMCC→OFDMリセット接続を切断（リセットループ防止）

**live_oneseg.py ワークアラウンド:**
- HackRF水晶誤差（~20PPM）を事前にRotatorで補正
- gr-isdbtの整数周波数オフセット処理に残るデータキャリア適用バグを回避
- `--calibrate` で自動校正、`--sc` で手動オフセット指定に対応
